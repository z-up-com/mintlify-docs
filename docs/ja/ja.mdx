### [​](#マルチプロバイダールーティングとエンタープライズ-sla)マルチプロバイダールーティングとエンタープライズ SLA

複数の LLM プロバイダー間でインテリジェントルーティングを内蔵し、高可用性と低レイテンシを確保。あるプロバイダーに問題が発生しても、リクエストは自動的にバックアッププロバイダーにルーティングされ、サービス中断なし。
### [​](#99-9%-可用性-sla)99.9% 可用性 SLA

自動フェイルオーバー付きのエンタープライズグレードの信頼性。マルチプロバイダールーティングで最小レイテンシでのリクエスト処理を保証。
### [​](#グローバル-cdn-高速化)グローバル CDN 高速化

世界中のエッジロケーションで最低レイテンシを実現。最適化されたルーティングで応答時間を短縮し、グローバルなユーザー体験を向上。

## [​](#sdk-とコード例)SDK とコード例

Python、Node.js、Java 用の公式 SDK。すべての SDK が OpenAI 互換で、Base URL 設定の変更のみでシームレスに移行可能。
### [​](#python-sdk)Python SDK

`pip install openai` でインストール。Base URL を設定し、既存の OpenAI コードをそのまま使用、変更不要。
コピー
```shiki
from openai import OpenAI
client = OpenAI(
    base_url="https://api.dashlyai.cc/v1",
    api_key="your-dashlyai-key"
)

```
### [​](#node-js-sdk)Node.js SDK

`npm install openai` でインストール。baseURL パラメータを DashlyAI エンドポイントに設定し、既存の統合を維持。
コピー
```shiki
import OpenAI from 'openai';
const client = new OpenAI({
  baseURL: 'https://api.dashlyai.cc/v1',
  apiKey: 'your-dashlyai-key'
});

```
### [​](#java-sdk)Java SDK

OpenAI 互換 Java クライアント。簡単な設定変更で全 DashlyAI モデルとプロバイダーにアクセス可能。
## [​](#openai-からの移行)OpenAI からの移行

5 分以内に DashlyAI に切り替え可能。コードの書き直し不要——Base URL と API Key の変更のみ。
1

API Key を取得

DashlyAI に登録し、[コンソールダッシュボード](https://dashlyai.cc/console/token)から API Key を取得。2

Base URL を更新

OpenAI Base URL を DashlyAI エンドポイントに変更。既存の SDK 統合はそのまま動作。3

エンドポイントをテスト

新しい Base URL で既存コードを実行。すべての OpenAI 互換エンドポイントは同じ形式で応答。コンソールダッシュボードでレイテンシとレート制限を確認。

## [​](#よくある質問)よくある質問

OpenAI から移行するには？

Base URL を `https://api.dashlyai.cc/v1` に変更し、DashlyAI API Key を使用するだけです。既存の SDK をそのまま使用可能——コードの書き直しは不要。移行は通常 5 分以内に完了します。

SLA に含まれるものは？

エンタープライズ SLA には 99.9% 可用性保証、グローバル CDN 高速化、自動フェイルオーバー付きマルチプロバイダールーティング、リアルタイムステータス監視が含まれます。レート制限管理と Webhook サポートも含まれます。

OpenAI と比較した料金は？

DashlyAI は公式プロバイダーより手頃な透明なトークン単価を提供。サブスクリプション不要の従量課金、エンタープライズ利用でボリューム割引、隠れた費用なし。

レイテンシとパフォーマンスは？

グローバル CDN 高速化とインテリジェントマルチプロバイダールーティングにより低レイテンシを保証。世界中のエッジロケーションで最適な応答時間を確保。リアルタイムステータス監視でパフォーマンスを追跡可能。[汎用チャットAPI](/ja/api-reference/texts/general/chat-completions)⌘I
