# Настройка Cherry Studio

> Используйте ToAPIs в качестве бэкенда API для Cherry Studio

## Подготовка

Перед началом убедитесь, что у вас есть:

1. **Установленный Cherry Studio**

   Скачайте и установите Cherry Studio с [официального сайта](https://cherry-ai.com)

2. **API-ключ ToAPIs**

   Войдите в [консоль ToAPIs](https://toapis.com/console/token) для получения API-ключа (начинается с `sk-`)

**Примечание:** Если у вас ещё нет аккаунта ToAPIs, сначала зарегистрируйтесь на [ToAPIs](https://toapis.com).

## Шаги настройки

### Шаг 1: Открыть настройки провайдеров моделей

1. Запустите Cherry Studio
2. Нажмите на иконку **Настройки** (шестерёнка) в левом нижнем углу
3. Перейдите в раздел **Модельные сервисы** или **Провайдеры**

### Шаг 2: Добавить ToAPIs как провайдера

1. Нажмите кнопку **+** в правом верхнем углу списка провайдеров
2. В появившемся диалоге:
   * **Название**: введите `ToAPIs` (или любое другое имя)
   * **Тип провайдера**: выберите **New API** (совместимо с OpenAI)
3. Нажмите **Подтвердить**

### Шаг 3: Заполнить API-конфигурацию

На странице настройки провайдера ToAPIs введите следующие данные:

| Поле          | Значение                                |
| ------------- | --------------------------------------- |
| **API-ключ**  | Ваш API-ключ ToAPIs (`sk-xxxxxxxxxxxx`) |
| **API-адрес** | `https://toapis.com/v1/`                |

### Шаг 4: Добавить модели

1. На странице настройки провайдера нажмите **Добавить модель**
2. Заполните данные модели:
   * **ID модели**: введите ID нужной модели (см. рекомендуемые модели ниже)
   * **Тип эндпоинта**: выберите **OpenAI**
3. Сохраните

**Рекомендуемые модели:**

| Название модели   | ID модели                                                                   | Описание                                                                 |
| ----------------- | --------------------------------------------------------------------------- | ------------------------------------------------------------------------ |
| Claude Opus 4.6   | [`claude-opus-4-6`](/ru/api-reference/chat/claude-opus-4-6/chat)            | Наиболее мощная, для сложных задач                                       |

| Claude Haiku 4.5  | [`claude-haiku-4-5-20251001`](/ru/api-reference/chat/claude-haiku-4-5/chat) | Самая быстрая, для лёгких задач                                          |
| GPT-5             | [`gpt-5`](/ru/api-reference/chat/models)                                    | Новейшая флагманская модель OpenAI                                       |
| GPT-4o            | [`gpt-4o`](/ru/api-reference/chat/models)                                   | Высококачественные диалоги и мультимодальность                           |
| GPT-4o Mini       | [`gpt-4o-mini`](/ru/api-reference/chat/models)                              | Быстрая и экономичная                                                    |

Повторите шаги для добавления нескольких моделей.

### Шаг 5: Включить и проверить

1. Убедитесь, что переключатель рядом с провайдером **включён**
2. Нажмите **Проверить соединение** или **Тест**, чтобы убедиться в корректности настроек
3. Сообщение об успехе подтверждает, что всё настроено верно

## Начало работы

После настройки:

1. Вернитесь на главный экран Cherry Studio
2. В выборщике моделей в верхней части окна чата выберите добавленную модель ToAPIs
3. Начните общение

## Устранение неполадок

### Тест соединения не проходит?

1. Проверьте **API-ключ** — он должен начинаться с `sk-`
2. Убедитесь, что **API-адрес** задан как `https://toapis.com/v1/` (обратите внимание на завершающий слеш)
3. Проверьте активность ключа в [консоли ToAPIs](https://toapis.com/console/token) и убедитесь, что на счету достаточно средств
4. Проверьте подключение к интернету и доступность `https://toapis.com`

### Нужной модели нет в списке?

Cherry Studio использует ручное добавление моделей. Просто введите ID модели при добавлении. Все доступные модели можно найти в [списке моделей ToAPIs](https://toapis.com/console/models).

### Распространённые коды ошибок

| Ошибка                      | Причина                                    | Решение                         |
| --------------------------- | ------------------------------------------ | ------------------------------- |
| `401 Unauthorized`          | Недействительный или просроченный API-ключ | Создайте новый API-ключ         |
| `429 Too Many Requests`     | Превышен лимит запросов                    | Подождите немного и повторите   |
| `500 Internal Server Error` | Временная проблема на сервере              | Повторите через несколько минут |
| `insufficient_quota`        | Недостаточно средств на счету              | Пополните баланс в консоли      |
